Architecture

How the system fits together

The PSBN prototype is built as a heterogeneous network environment where Band 14 LTE operates alongside complementary access networks. This architecture supports the experimental evaluation of multipath protocols, particularly MPTCP, for public safety communication resilience.

Architecture diagram

A system-level view of how Band 14 LTE, complementary networks, and multipath-enabled endpoints connect in the prototype.

Core components

The platform is assembled from connectivity, gateway, and endpoint components within a PSBN testbed designed to evaluate communication behavior across multiple networks.

Band 14 PSBN access

Band 14 acts as a public safety anchor network within the testbed, providing a dedicated public safety path for evaluation.

Complementary access networks

Additional networks such as 5G and WiFi provide alternative communication paths, reflecting the heterogeneous conditions public safety users may encounter.

Multipath protocol layer

Multipath protocols, especially MPTCP, are used to evaluate how TCP-based applications can leverage multiple available paths for improved resilience.

Traffic flow (high-level)

A narrative that reflects how public safety applications interact with multiple networks in the testbed.

  1. An application starts from an endpoint using TCP-based communication.
  2. Multiple networks are available, including Band 14 LTE and complementary access paths.
  3. MPTCP manages subflows across these paths to evaluate continuity, reliability, and adaptability.
  4. A disruption occurs, such as link loss, handover, or path degradation.
  5. Application behavior is evaluated using evidence, captures, and planned continuity metrics.
