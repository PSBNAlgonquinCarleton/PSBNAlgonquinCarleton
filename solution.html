Workflow overview

The methodology is structured as a repeatable pipeline: establish a PSBN testbed, expose multiple network paths, evaluate multipath protocol behavior, then prepare controlled scenarios for continuity analysis.

1
Configure the PSBN testbed
Set up Band 14-enabled devices within a PSBN testbed environment and integrate complementary access networks so that heterogeneous connectivity is available for evaluation.
Band 14TestbedHeterogeneous networks
2
Evaluate multipath protocol behavior
Evaluate how multipath protocols, particularly MPTCP, allow TCP-based applications to use multiple interfaces and maintain session continuity across available paths.
MPTCPMultipathSession continuity
3
Prepare disruption scenarios
Define controlled events such as link loss, handover transitions, and degraded path conditions in order to study how applications behave when network availability changes.
Link lossHandoverDegradation
4
Collect evidence and publish
Once testing is carried out, evidence will be published using short videos, captures, and clear visual summaries such as plots and tables to show throughput, reliability, and adaptability outcomes.
VideosPlotsReporting

Planned scenarios

Scenarios are designed to demonstrate how TCP-based applications behave across multiple network paths in a public safety setting.

Link loss during active session

Disable one available path while a TCP-based workflow is active and observe continuity behavior.

Handover transition

Trigger a change between available networks and observe stability, adaptability, and session usability.

Degraded path conditions

Introduce reduced bandwidth or increased latency on one path and observe how traffic adapts across networks.
