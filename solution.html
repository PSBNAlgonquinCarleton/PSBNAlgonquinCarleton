Workflow overview

The methodology follows a structured pipeline: configure the PSBN testbed, expose multiple network paths, evaluate multipath protocol behavior, and analyze continuity under controlled disruptions.

1
Configure the PSBN testbed
Band 14-enabled devices are configured within a PSBN testbed and integrated with complementary access networks, enabling heterogeneous connectivity for evaluation.
Band 14TestbedMulti-network
2
Evaluate multipath protocol behavior
Multipath TCP (MPTCP) enables TCP-based applications to utilize multiple interfaces simultaneously, maintaining session continuity across available network paths.
MPTCPMultipathContinuity
3
Apply disruption scenarios
Controlled events such as link loss, handovers, and degraded network conditions are introduced to observe how active sessions behave under changing connectivity.
Link lossHandoverDegradation
4
Analyze and present results
Results are documented using system logs, visual captures, and performance summaries such as throughput stability and session continuity.
LogsMetricsVisualization

Evaluation scenarios

These scenarios demonstrate how TCP-based applications behave across multiple network paths in a public safety environment.

Link loss during active session

One network path is removed while a session is active to evaluate continuity behavior.

Handover transition

Devices transition between available networks while maintaining session usability.

Degraded path conditions

Bandwidth and latency are modified to observe how traffic adapts across paths.
