Key demonstrated results

The platform demonstrates working transport-layer continuity concepts, wireless network activity, application-layer visibility, and messaging functionality across connected devices.

MPTCP continuity

MPTCP has been demonstrated as part of the continuity workflow, supporting the project’s transport-layer resilience objective.

Network connectivity

Amarisoft logs and connected-device behavior show successful network-side communication and device interaction.

Operational awareness

TAK / WebTAK services provide a shared live view, including client visibility, map-based tracking, and server-side monitoring.

Amarisoft device and attach activity

The Amarisoft environment shows connected-device interaction and signaling activity within the wireless testbed.

Connected device view

A device interacting with the Amarisoft platform, showing practical end-device participation in the network environment.

Signaling and attach logs

Amarisoft log output showing network-side activity such as signaling, registration flow, and session-related events.

Result Wireless platform evidence

Together, these results confirm that the wireless platform is functioning as more than a diagram or simulation: it supports real device interaction, real logs, and observable system behavior.

TAK integration and shared operational view

The platform also demonstrates application-layer functionality through TAK services, including server metrics, client visibility, and a live shared map interface.

TAK Server metrics dashboard

The metrics dashboard provides server-side visibility into uptime, connected clients, resource usage, and system health.

WebTAK shared map

WebTAK provides a live operational view where connected entities can be seen on a shared map environment.

Terminal-to-device messaging

A message was sent through the terminal to devices on the network, demonstrating application-layer communication reaching connected endpoints.

Overall system significance

The combined results show that the PSBN Prototype is not limited to isolated subsystems. It demonstrates continuity concepts, wireless activity, server-side infrastructure, shared situational awareness, and end-device messaging within one integrated capstone platform.

  • ✓ MPTCP continuity behavior demonstrated in the platform workflow
  • ✓ Wireless platform activity verified through Amarisoft system logs
  • ✓ TAK / WebTAK integration provides real-time shared operational visibility
  • ✓ Terminal-based messaging successfully reached devices on the network
Summary Integrated result

The results support the project’s main objective: building a practical PSBN-oriented resilience platform that connects communication-layer functionality with public safety application-layer tools.
